# Ethics

SomaGuard is a defensive, simulation-only research project. Its purpose is to study how human control, consent, and safety boundaries can be represented around body-adjacent device concepts.

It is not a project for controlling real devices.

## Defensive Scope

The project is limited to documentation and conceptual architecture. It must not include code, dependencies, APIs, user interfaces, real hardware interaction, medical device control, implant logic, malware, exploit logic, offensive behavior, or real body manipulation.

All examples should remain fictional, simulated, and non-operational.

## Human Sovereignty

Human sovereignty means the person remains the primary authority over body-adjacent capability.

In this project, that means:

- Consent should be explicit and revocable.
- Permission should be limited to clear purposes.
- Safety modes should be understandable.
- Emergency locks should prioritize human control.
- Audit logs should explain decisions in plain language.
- Ambiguous authority should lead to denial or review.

The architecture should never normalize hidden control over a person or their body-adjacent environment.

## Misuse Boundaries

This project must not provide instructions or structures that enable misuse.

Out of scope:

- Malware or exploit behavior
- Offensive device control
- Bypassing consent
- Bypassing safety modes
- Real hardware access
- Real medical workflows
- Implant interaction
- Covert surveillance
- Real body manipulation

If a future idea creates ambiguity, it should be rewritten at a higher level of abstraction or removed.

## No Offensive Usage

SomaGuard is intended for defensive research, safety planning, education, and architecture review.

It should help people ask responsible questions:

- Who has authority?
- What did the human consent to?
- What happens when trust is unclear?
- How can a system fail safe?
- How can a human stop the system?
- How can decisions be reviewed later?

The project should remain calm, precise, and safety-oriented. It should avoid hype, fear, and operational details that could support harmful use.
